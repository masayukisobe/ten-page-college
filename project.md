# 10ページ大学 project

## 企画の核

10ページ大学は、AI時代の大人の学び直しシリーズである。

ブランドコピーは、**「知ってる言葉を、説明できる教養へ。」**

読者がニュース、仕事、日常会話、SNSで見聞きしている言葉を、仕組みまで少し説明できる状態へ近づける。専門書や長い講義へ入る前に、入口、見取り図、問い方を渡す。

本文の価値は、読者の見え方を変えることにある。

10ページ大学の商品価値は、高度な内容を、読者の普段の経験から届く例、比喩、図解で説明できる形へ落とすことにある。すでに易しい内容を易しく言うだけで満足せず、読者が聞いたことのある言葉の裏側にある深い仕組みまで踏み込む。

「知ってる言葉を、説明できる教養へ。」は、名前を知る段階から、仕組みを自分の言葉で説明できる段階へ進むことを指す。そのため、本文では用語の紹介より、用語の背後にある設計、制約、トレードオフ、歴史的な理由、日常の見え方とのつながりを重視する。

読後に目指す感覚:

> 名前だけは知っていたけど、そういう仕組みだったのか。ちょっと面白い。

## 読者像

想定読者は、仕事や生活の段取りはできるが、専門的な抽象化や数式には慣れていない大人である。

読者は、AI、半導体、クラウド、文字コード、圧縮、OS、ネットワークのような言葉を、見聞きした経験がある。

本文では、読者の普段の経験、画面で見ているもの、手元で起きる操作から入り、その言葉の裏側を説明する足場を作る。

## 商品としての形

1テーマにつき、本文10ページ程度の電子コンテンツを作る。

「10ページ」は、学びの最小単位を示す編集思想である。

標準構成:

1. はじめに
2. 本文10ページ
3. 重要用語
4. 理解確認クイズ
5. ChatGPT深掘りプロンプト
6. 次に検索するキーワード

本文ページは、図や例示枠を除いた本文だけで平均50行程度、約1000字程度を目安にする。図は1ページ4-5点を標準にする。

## 執筆方針

本文は、ですます調で書く。

読者を子ども扱いせず、知識の入口にいる大人として扱う。

専門語は、必要な場面で使う。ページタイトルにも、本文の説明トーンに合う専門語を使ってよい。目次に少し硬い言葉が並び、本文を読むと説明できるようになることが、ブランドコピーとつながる。

UTF-8、RGB、JPEG、圧縮、コーデックのように、読者が普段目にする語の仕組みを説明できる場合は積極的に使う。名前を出したら、その語が何の約束か、何を変換しているのかを本文で一口ぶん説明する。

本文の面白さは、具体例、順番、比喩、図、短い回収で作る。

比喩や身近な例は、低いレベルの説明を飾るためではなく、高度な内容へ読者を連れていく橋として使う。フォント、文字コード、圧縮、コーデック、マルチモーダルのように、聞いたことはあるが説明しにくい語ほど、10ページ大学らしい価値を出せる。

専門用語のラベルを付けるだけで、理解が上がったように見せない。HTTP、DNS、TCP、クラウドのような語は、それが何と呼ばれるかより、なぜその形が必要になったのか、どの制約を受けているのか、何を捨てて何を得ているのかまで踏み込む。読者が常識として知っている説明に専門名を貼っただけなら、商品価値は上がらない。

「なぜ必要か」は、読者が普段見る画面、設定、契約、待ち時間、トラブル、同じボタン操作の違いから立て、その後で設計上の制約へ進む。技術側の事情だけで理由を書くと、正しくても教科書的に見える。

仕組みや構造を表す名詞を説明するときは、読者がその存在を知っているかを先に見る。OSやアップデートのように存在を知っている語は、「それがない場合に何が困るか」から必要性を立てる。イベントやプロセスのように存在自体を知らない語は、「それがあると何ができるか」「何をまとめて扱えるか」から入り、名前と役割へ進める。

読者がすでに知っている常識を言い換えるだけの文は、本文価値を下げる。常識を入口にする場合も、必ず仕組み、分解軸、制約、読後の見え直しへ進める。

本文では、説明上必要な場合を除き、「Xではない」「Xとは限らない」という否定命題から入らない。読者がまだ持っていない誤解を先に提示すると、本文側で余計なハテナを作る。誤解防止が必要な場合も、まず正の構造を置き、そのあとで範囲や限界を短く示す。

身近な商品名、契約名、設定画面の言葉は、読者にとって一体に見えやすい。たとえばスマホの通信プラン、光回線、Wi-Fi、プロバイダ、クラウド保存のように、申し込みや請求ではまとめて見えるものでも、裏側では役割が分かれていることがある。本文では、読者がすでに知っている表面を長く説明せず、その表面の裏にある分担、接続先、制約、トラブル時の切り分けへ進める。

## リポジトリ構成

主要な作業場所:

- `project.md`: プロジェクト全体の現在方針。
- `series_plan.md`: 情報学部のシリーズ計画。
- `handoff.md`: 現在の作業状態と引き継ぎ。
- `policies/`: 執筆・レビュー方針と観点文書。
- `series/semicon-computer/01/`: 第1巻。
- `series/semicon-computer/02/`: 第2巻。

各巻の `structure.md` は、その巻の構成、ページタイトル、説明範囲を管理する。

各巻の `draft.md` は、個別 `draft-0x.md` を組み上げた確認用として使う。

# abstract-05

## ページタイトル

音声データ：波形を時間ごとの数値として記録する

## 目標ボリューム

- 本文だけで50行程度、約1000字程度を目安にする。
- 図は4点を標準にする。
- 4ページ目の標本化と量子化を、音声で具体化する。

## 構成、節見出し、行数目安

### 1. 音は空気の震え: 8-10本文行
- 本文節見出し: 音は空気のゆれから始まる

- 音は、空気の震えとして伝わる。
- マイクは、その震えを電気信号に変える。
- 図: [ここに「人の声 → 空気の震え → マイク → 電気信号」の図を入れる]

### 2. 波形を測る: 12-14本文行
- 本文節見出し: 波形を時刻ごとの数字にする

- 電気信号の変化を、時間ごとに測る。
- 測った点が数字の列になる。
- 音声ファイルは、音そのものではなく、音の変化を記録した数字列である。
- 数字の列だけを見ると波形は見えないが、時刻ごとの高さを順番に持っていると説明する。
- 図: [ここに「音の波形を点で測って数字列にする」図を入れる]

### 3. サンプリング周波数とビット深度: 12-14本文行
- 本文節見出し: 測る回数と目盛りの細かさ

- サンプリング周波数は、1秒間に何回測るか。
- ビット深度は、音の大きさをどれくらい細かく記録するか。
- 44.1kHzは1秒に44100回測ること、16bitは波形の高さを65536段階で読むこととして説明する。
- 図: [ここに「サンプリング周波数とビット深度」の図を入れる]

### 4. 数字から音へ戻す: 8-10本文行
- 本文節見出し: 数字から音へ戻す

- 保存してある数値を順番に読み、スピーカーの動きへ戻す。
- 録音は波形を数値にすること、再生は数値から波形を作ることとして整理する。
- 図: [ここに「録音は波形を数字にし、再生は数字から波形を作る」図を入れる]

### 5. 早回しと時間軸: 10-12本文行
- 本文節見出し: 早回しで声が高く聞こえる理由

- 早回しで声が高くなる例を使う。
- 同じ波形を短い時間に押し込むと、変化が速くなり、高く聞こえる。
- 図: [ここに「同じ波形を短い時間に押し込むと波の山の間隔が短くなる」図を入れる]

### 6. 回収: 4-6本文行
- 本文節見出し: 音も約束どおりに読み戻す

- 音声データは、時間に沿って並ぶ数字の列である。
- 音を扱うには、時間の間隔と音量の細かさが大事になる。

## 中身や仕組みがわかった楽しさ

- 音声ファイルが、音の変化を時間ごとに記録したものとして見える。
- サンプリング周波数やビット深度が、カタログ用語ではなく測り方の言葉になる。
- 早回しの声の変化が、時間軸の操作として説明できる。

## 構造分析とシーン比喩

- 概念・用語・仕組み: サンプリング周波数、ビット深度、44.1kHz、16bit、録音、再生。
- 役割: サンプリング周波数は1秒に測る回数、ビット深度は高さを読む目盛りの細かさ。
- 情報の流れ: 空気のゆれ -> マイクの波形 -> 時刻ごとの数字 -> スピーカーの動き -> 空気のゆれ。
- 時間順の変化: 録る、数字にする、保存する、順番に読む、鳴らす。
- 採用するシーン比喩: 1秒間の足跡と高さの目盛り。44.1kHzは1秒に44100個の足跡、16bitは高さを細かい段階で読む目盛りとして扱う。
- 本文反映: サンプリング周波数とビット深度の節に、44.1kHzと16bitの見方を短く入れる。

## 本文で避けること

- サンプリング周波数とビット深度を混同しない。
- ハイレゾや可聴域に広げすぎない。

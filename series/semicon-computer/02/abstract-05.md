# abstract-05

## ページタイトル

音声データ：波形を時間ごとの数値として記録する

## 目標ボリューム

- 本文だけで50行程度、約1000字程度を目安にする。
- 図は4点を標準にする。
- 4ページ目の標本化と量子化を、音声で具体化する。

## 構成と行数目安

### 1. 音は空気の震え: 8-10本文行

- 音は、空気の震えとして伝わる。
- マイクは、その震えを電気信号に変える。
- 図: [ここに「人の声 → 空気の震え → マイク → 電気信号」の図を入れる]

### 2. 波形を測る: 12-14本文行

- 電気信号の変化を、時間ごとに測る。
- 測った点が数字の列になる。
- 音声ファイルは、音そのものではなく、音の変化を記録した数字列である。
- 図: [ここに「音の波形を点で測って数字列にする」図を入れる]

### 3. サンプリング周波数とビット深度: 12-14本文行

- サンプリング周波数は、1秒間に何回測るか。
- ビット深度は、音の大きさをどれくらい細かく記録するか。
- 図: [ここに「サンプリング周波数とビット深度」の図を入れる]

### 4. 早回しと時間軸: 10-12本文行

- 早回しで声が高くなる例を使う。
- 同じ波形を短い時間に押し込むと、変化が速くなり、高く聞こえる。
- 図: [ここに「早回しで時間軸が縮み声が高くなる」図を入れる]

### 5. 回収: 4-6本文行

- 音声データは、時間に沿って並ぶ数字の列である。
- 音を扱うには、時間の間隔と音量の細かさが大事になる。

## 中身や仕組みがわかった楽しさ

- 音声ファイルが、音の変化を時間ごとに記録したものとして見える。
- サンプリング周波数やビット深度が、カタログ用語ではなく測り方の言葉になる。
- 早回しの声の変化が、時間軸の操作として説明できる。

## 本文で避けること

- 音声ファイルの中に音そのものが入っているように書かない。
- サンプリング周波数とビット深度を混同しない。
- ハイレゾや可聴域に広げすぎない。

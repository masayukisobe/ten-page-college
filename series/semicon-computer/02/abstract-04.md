# abstract-04

## ページタイトル

標本化と量子化：信号を数字列に変換する

## 目標ボリューム

- 本文だけで50行程度、約1000字程度を目安にする。
- 図は4点を標準にする。
- 音声・画像・センサー値に共通する、現実の変化を数字にする考え方を扱う。

## 構成、節見出し、行数目安

### 1. 現実の変化はなめらかに見える: 8-10本文行
- 本文節見出し: なめらかな変化を測る

- 音の波、光の明るさ、温度の変化などを例にする。
- それらは物理的な信号として扱える。
- デジタルデータの設計には、物理的な変化をどう測るかが関わる。

### 2. 標本化は点で測ること: 12-14本文行
- 本文節見出し: いつ測るかを決める

- 標本化（サンプリング）は、一定間隔で値を測ること。
- なめらかな線を、点の並びにする。
- 間を飛ばして測るため、細かすぎる変化は落ちる。間隔を細かくすると、変化を追える点が増える。
- 図: [ここに「なめらかな波形を等間隔の点で測る」図を入れる]

### 3. 量子化は段階に丸めること: 12-14本文行
- 本文節見出し: どの目盛りで読むかを決める

- 測った値を、決まった段階の数字にする。
- ものさしの目盛りが細かいほど、細かく記録できる。
- 横方向が標本化、縦方向が量子化として見える図で分ける。
- 図: [ここに「測った値を段階的な数字に丸める」図を入れる]

### 4. 信号処理という見方: 12-14本文行
- 本文節見出し: 細かく残すほど重くなる

- 信号処理は、変化するものを測って、整えて、扱いやすくする技術として説明する。
- 音だけではなく、光や温度にも同じ見方が使える。
- 体温計やものさしのように、読者が知っている測定の目盛りを使い、説明の密度を上げる。
- 細かく測るほどデータ量が増え、粗く測るほど細部が落ちる設計として見せる。
- 図: [ここに「現実の信号 → 標本化 → 量子化 → 数字列」の図を入れる]
- 図: [ここに「音・光・温度が信号として数字になる」図を入れる]

### 5. 回収: 4-6本文行
- 本文節見出し: 世界の変化を数字に切り分ける

- デジタルデータは、現実から切り離された記号だけではない。
- 測る、区切る、数字にするという設計で現実とつながっている。

## 中身や仕組みがわかった楽しさ

- サンプリングや量子化が、急に身近な「測る」行為として見える。
- 音、光、温度が同じ信号の仲間として見える。
- デジタルデータが、物理世界を切って数字にしたものだとわかる。

## 構造分析とシーン比喩

- 概念・用語・仕組み: 標本化、量子化、測る間隔、目盛り、粗さによる失敗。
- 役割: 標本化はいつ測るか、量子化はどの目盛りで読むかを決める。
- 情報の流れ: なめらかな信号 -> 時刻ごとの点 -> 近い目盛りの値 -> 数字列。
- 時間順の変化: 現実の変化を見る、測る時刻を決める、値を目盛りへ丸める、数字列として扱う。
- 失敗パターン: 測る間隔が粗いと速い変化を拾いそこねる。目盛りが粗いと段差が目立つ。
- 採用するシーン比喩: 防犯カメラのコマ送りと体温計の小数表示。撮る間隔と表示の目盛りが、残る情報を決める。
- 本文反映: 粗い標本化と粗い量子化で何が起きるかを、失敗例として短く加える。

## 本文で避けること

- デジタル化すると必ず悪くなると単純化しない。
- サンプリング定理に深入りしない。
- 信号処理を音声だけの話にしない。

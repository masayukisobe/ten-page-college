# abstract-01

## ページタイトル

文字も画像も、最後は一列に並べられる

## 目標ボリューム

- `draft-01.md` は、本文だけで50行程度、約1000字程度を目安にする。
- ここでいう本文行は、ページ見出し、空行、図指定、コードブロックや `text` ブロックなどの例示枠を除いた、読者が本文として読む文章の行である。
- 図は4-5点を標準にする。1ページ目は現状4点で成立するが、必要なら5点にしてよい。
- `abstract-01.md` は本文の言い回しを固定しない。再現すべきなのは、説明順、図示内容、読者の理解の更新、行数配分である。
- 再執筆時は、同じ内容を言い換えて厚くせず、説明の橋が抜けていないかを優先して見る。

## 構成と行数目安

### 1. 第1巻のおさらい: 6-8本文行

- 半導体スイッチのオフとオンを、0と1として読むことを確認する。
- 0か1のどちらか1個ぶんの情報を、ビットと呼ぶことを短く戻す。
- 図: [ここに「半導体スイッチがオフなら0、オンなら1になる」図を入れる]
- 楽しさ: 第1巻のスイッチの話が、文字や画像の話につながる。

### 2. 文字は番号で表せる: 10-12本文行

- 「あいうえお表」のように、文字へ番号をつければ、数字の並びを文字として読める。
- 実際のコンピュータでは、ひらがな、カタカナ、漢字、アルファベット、記号、絵文字まで番号をつけて扱う。
- 文字に番号をつけ、その番号を0と1の並びとして保存し、画面に出すときに文字へ戻す。
- 図: [ここに「あいうえお表で、数字の並びが文字として読める」図を入れる]
- 楽しさ: 文字の形そのものではなく、番号と表で文字が戻るとわかる。

### 3. 文や文章も順番に並べられる: 14-16本文行

- 文字を説明したあと、文や文章へ進む。
- 文は文字の並びであり、文字は番号で表せるので、1行の文も番号の並びで表せる。
- 2行の文章は、1行の文のように順番に並べて扱える。
- ただし、どこで行が変わるのかを示す必要がある。
- 改行も「ここで行を変える」という印として入り、文字と同じように番号を持つ。
- 説明の橋は「2行の文章 -> 1行の文のように見る -> 文は文字の並び -> 文字は番号で表せる -> 文も番号の並びで表せる -> 文章をデジタルデータとして表現できる」の順にする。
- 図: [ここに「2行の文章 → 文字と改行を順番に並べる → 0と1の列」の図を入れる]
- 楽しさ: 画面では改行に見えるものまで、ファイルの中では番号として並んでいるとわかる。

### 4. 画像も順番に並べられる: 14-16本文行

- 白黒のドット絵から始め、白を0、黒を1と決める。
- 上の行から左、右へ順番に読めば、絵も0と1の並びにできる。
- 一列にしただけでは絵に見えないが、横幅と縦幅が決まっていれば、決まった数で折り返して絵に戻せる。
- 縦横サイズは、ただの飾りではなく、一列の数字を絵に戻すための情報である。
- 図: [ここに「白黒ドット絵 → 上から順に0と1へ読む → 横幅で折り返して絵に戻る」図を入れる]
- 楽しさ: 絵が、色の数字を決まった幅で折り返したものとして見え直す。

### 5. 回収: 6-8本文行

- 文字も画像も、画面では平面に見えても、コンピュータの中では順番に並んだ数値として扱える。
- 文字は、文字や改行につけた番号を順番に読む。
- 画像は、縦横のサイズを決めて、色の数字を順番に読む。
- 見えている形そのものを保存するのではなく、数字の並びを、読み方の約束で文字や画像に戻していると締める。

## 中身や仕組みがわかった楽しさ

- 文字も画像も、見た目はまったく違うのに、コンピュータの中では数値の並びとして扱えるとわかる。
- 改行までデータとして入っていると知ると、ふだん見ている文章の裏側が少し違って見える。
- 白黒のドット絵を上から順番に読んでいくと、絵が0と1の並びになるのが直感的にわかる。
- 縦横サイズがわかるだけで、一列に並んだ0と1を絵に戻せるところに、コンピュータの約束事のおもしろさがある。

## 本文で避けること

- 「この巻では」「第2巻で大事なのは」のような案内文で水増ししない。
- 「小さな疑問が出ます」のような制作意図を本文に出さない。
- 「直列化」という語は使わず、「一列に並べる」「順番に並んでいる」のように説明する。

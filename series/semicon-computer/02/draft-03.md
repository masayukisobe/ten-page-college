# 3ページ目：ピクセルとRGB：画像は色の数値表でできている

## 写真を小さな点で見る

文字は、番号の並びとして表せました。

では、画像はどうでしょう。

写真やイラストは、文字よりずっと複雑に見えます。

文字を番号で表したように、画像も小さな点の数字で表せます。

画面を大きく拡大すると、小さな四角が並んでいます。

その1つ1つを、ピクセルと呼びます。

画像は、ピクセルを縦横に並べたものです。

![写真を拡大するとピクセルの集まりに見える](images/fig007-photo-pixels-zoom.png)

## 白黒から色へ

1ページ目では、白黒のドット絵を使いました。

白なら0。

黒なら1。

それだけでも、簡単な絵は作れます。

ただし、ふつうの写真は白黒だけでは足りません。

赤い夕焼けもあります。

青い空もあります。

肌の色も、服の色も、影の色もあります。

そこで、1つのピクセルに色の数値を持たせます。

## RGBは光の強さの数字

このページでは、色をRGBという3つの数字で見ます。

RGBは、赤、緑、青の光の強さです。

赤を英語でRed。

緑をGreen。

青をBlue。

その頭文字でRGBです。

![1つのピクセルがR・G・Bの3つの数値を持つ](images/fig008-rgb-values-per-pixel.png)

たとえば、赤だけ強くすれば赤く見えます。

緑だけ強くすれば緑です。

青だけ強くすれば青です。

3つを全部強くすれば白に近づきます。

全部弱くすれば黒に近づきます。

実際には、強さを数字で持ちます。

たとえば、`255, 0, 0` なら赤です。

`0, 255, 0` なら緑です。

`0, 0, 255` なら青です。

`255, 255, 255` なら白です。

`0, 0, 0` なら黒です。

![RGBの数値を変えるとピクセルの色が変わる](images/fig008b-rgb-color-examples.png)

ここでも、数字だけで終わりません。

RGBの数字を、どんな色として光らせるかという約束があります。

その代表的な約束の1つが、sRGBです。

Webやスマホの画像では、sRGBを前提にすることが多くあります。

同じ `255, 0, 0` でも、どの色の約束で読むかがそろうから、画面上で赤として扱えます。

## 画像は色の数値表

画像全体は、この色の数値表です。

横に何個、縦に何個のピクセルがあるかを決めます。

この縦横の細かさを、解像度と呼びます。

同じ写真でも、細かいマスで持てばなめらかに見えます。

大きなマスで持てば、粗い画像に見えます。

そのかわり、細かいほど数字の数は増えます。

その1つ1つに、RGBの数字を置きます。

左上から右へ、順番に読んでいきます。

1行目が終わったら、2行目を読みます。

それを最後の行まで続けます。

すると、画像も数字の並びとして持てます。

見た目は平面でも、中身は順番に読めるのです。

![ピクセル表を左上から順番にRGBの数字列として読む](images/fig008c-pixel-grid-rgb-stream.png)

## フィルターは数値を変える

スマホのフィルターも、この考え方で見えます。

セピア調にすると、写真が古い紙のような色になります。

でも、写真の中で小さな職人が色を塗っているわけではありません。

ピクセルのRGBの数字を、別の数字へ変えているのです。

赤みを少し増やす。

青みを少し減らす。

明るさを少し変える。

そういう計算を、たくさんのピクセルに行います。

ひとつのピクセルで見れば、元のRGB値が、別のRGB値に置き換わります。

![元の1ピクセルのRGB値をセピア調のRGB値に変える](images/fig009-rgb-to-sepia-pixel.png)

## 写真も数字の表として戻る

写真は、絵の形のまましまわれているわけではありません。

縦横に並んだピクセルがあります。

それぞれのピクセルに、色の数字があります。

その数字を読み直すと、画面に写真が戻ってきます。

## 構成、節見出し、行数目安

### 1. データはファイルとして扱われる: 8-10本文行

- 第2巻で扱った文字、画像、音声、動画をファイルへつなげる。
- ファイルはデータに名前、形式、場所を与える単位と説明する。

### 2. ファイルには名前と形式がある: 10-12本文行

- `memo.txt` や `photo.jpg` のような例から形式の手がかりを説明する。
- 形式はデータをどう読むかの約束だと第2巻へ戻す。

### 3. フォルダとパスは場所を示す: 10-12本文行

- ファイルには、あとでたどるための場所がある。
- フォルダは見た目の入れ物だが、技術的には名前や場所をたどるための管理情報である。
- パスは住所。
- どの順番でたどればファイルへ行けるかを説明する。

### 4. ファイルシステムが管理する: 12-14本文行

- ファイルシステムは、データを保存し、名前や場所で探せるようにする仕組み。
- ストレージは、SSDやHDDのように0と1を長く残す記憶装置である。
- ファイルシステムは、ストレージ上のデータに名前、場所、サイズなどの管理情報を与えるOS側の仕組みである。
- フォルダは物理的な箱ではなく、ファイル名や場所をたどるための管理情報として概要レベルで扱う。
- 図書館の棚番号と目録を比喩に使う。

### 5. 見つからないは、たどれないということ: 10-12本文行

- 画面上の「見つからない」は、ストレージ全体を目で探した結果ではなく、今の手がかりでたどれない状態だと説明する。
- 名前、パス、権限、形式のずれで「見つからない」「開けない」が起こると整理する。
- 保存とは、データを記録し、あとでたどれるようにすることだと締める。

## 説明すること

- ファイルは、データに名前、形式、場所を与える単位である。
- フォルダはファイルをまとめる構造として見える。
- パスはファイルの住所である。
- ファイルシステムは、ファイルを保存し、探せるように管理する仕組みである。
- ストレージは記憶装置、ファイルシステムは名前と場所でたどるための管理方法である。

## 構造分析とシーン比喩

- 概念・用語・仕組み: ファイル、フォルダ、パス、ファイルシステム、形式、拡張子、権限。
- 理解上昇の入口（未知寄り）: ファイルシステムがあることで、0と1の並びに名前や場所を共通に与え、アプリがあとから同じ方法でたどれると説明する。
- 役割: ファイルは名前つき書類、フォルダは引き出し、パスは住所、ファイルシステムは目録と棚。
- 情報の流れ: アプリの保存依頼 -> OS -> ファイルシステム -> 保存場所へ記録 -> パスや名前で再取得。
- 時間順の変化: 作る -> 名前をつける -> 場所へ置く -> あとで探す -> 開く。
- 失敗パターン: ファイルを画面上のアイコンそのものとして扱うと、形式や権限の説明へ進めない。
- 類比できる日常シーン: 図書館、住所、書類の引き出し、目録。
- 採用するシーン比喩: ファイルシステムは図書館の棚番号と目録。比喩のあと、技術的にはOSが名前、場所、形式などを管理すると戻す。

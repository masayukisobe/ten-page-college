# abstract-09

## ページタイトル

権限とサンドボックス：アプリの使える範囲を区切る

## 目標ボリューム

- `draft-09.md` は本文だけで50行程度、約1000字程度を目安にする。
- 図は5点。許可画面、権限の鍵、サンドボックス、ファイル選択、便利さと安全のトレードオフを置く。

## 構成、節見出し、行数目安

### 1. 許可画面には意味がある: 8-10本文行

- 写真、カメラ、位置情報、マイクの許可画面から入る。
- OSがアプリの使える範囲を区切る場面として権限を定義する。

### 2. 権限は鍵のようなもの: 10-12本文行

- 権限を部屋や倉庫に入る鍵として説明する。
- 鍵を渡せば完全安全ではないが、範囲を区切れると扱う。

### 3. サンドボックスは区切られた作業場所: 12-14本文行

- アプリが何でも自由に見られる便利さと危険を置く。
- サンドボックスを、柵で区切られた作業場所として説明する。

### 4. OSが安全な受け渡しを用意する: 12-14本文行

- 写真選択、ファイル選択、共有ボタンを例にする。
- OSの受付を通して選んだものだけ渡す仕組みを説明する。

### 5. 区切るから不便に見えることもある: 10-12本文行

- 権限やサンドボックスは完璧な安全ではないが、重要な区切りである。
- 許可画面は役割分担が表に出たものだと締める。

## 説明すること

- 権限は、アプリがどの機能やデータを使ってよいかを決める約束である。
- サンドボックスは、アプリの動ける範囲を区切る考え方である。
- ファイル選択画面や共有ボタンは、安全な受け渡しの窓口になりうる。
- 便利さと安全にはトレードオフがある。

## 中身や仕組みがわかった楽しさ

- 許可画面が、邪魔な確認ではなく、アプリとOSの役割分担として見える。
- 同じ写真でもアプリによって見え方が違う理由を、権限や区切りで説明できる。
- 便利さと安全が、単純な良し悪しではなくトレードオフとして見える。

## 構造分析とシーン比喩

- 概念・用語・仕組み: 権限、サンドボックス、写真選択、ファイル選択、共有、位置情報、カメラ、マイク。
- 理解上昇の入口（混合）: 許可画面は既知寄りなので、区切りがない場合の危険から入る。サンドボックスは未知寄りなので、アプリごとに動ける範囲を区切れる効用として説明する。
- 役割: アプリは使いたい側、OSは範囲を決める側、権限は鍵、サンドボックスは区切られた場所。
- 情報の流れ: アプリの要求 -> OSの確認 -> ユーザーの許可 -> 範囲内のデータや機能を利用。
- 時間順の変化: 使いたいと頼む -> 許可画面が出る -> 許可される -> 選ばれた範囲だけ使う。
- 失敗パターン: 権限を絶対安全の保証として持ち上げると、判断の難しさが消える。
- 類比できる日常シーン: 鍵、受付、柵で区切られた作業場、倉庫から必要な荷物だけ渡す。
- 採用するシーン比喩: 権限は鍵、サンドボックスは柵で区切られた作業場所。比喩のあと、技術的にはOSがアプリの使える範囲を制御すると戻す。
